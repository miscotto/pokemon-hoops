New Routing Structure

Converting state-based navigation to real URL routes

Public Routes (no auth required)

/ — Home page (unchanged)
/tournaments — All tournaments list (waiting, active, completed)
/tournaments/[id] — Tournament bracket + results (spectate)
/users/[id] — Player profile: stats, roster, tournament history

Auth-Required Routes

/dashboard — Roster hub (list, create, manage rosters)
/rosters/[id]/build — Roster builder (owner only)
/profile — Redirects to /users/[yourId]
/admin — Admin panel (unchanged)

What Changes in /dashboard

Before (state-based)
setView("dashboard") → RosterDashboard
setView("builder") → RosterBuilder
setView("tournament") → TournamentView
setView("live-tournament") → LiveTournament
After (URL-based)
/dashboard → RosterDashboard
/rosters/[id]/build → RosterBuilder
/tournaments/[id] → TournamentView
/tournaments/[id] → LiveTournament (same page, different status)

Does this look right?

Note: TournamentView and LiveTournament merge into one /tournaments/[id] page — it shows simulation bracket or live bracket based on the tournament's status in the DB.
